# Schema Diff Package

The `schemadiff` package provides comprehensive schema comparison, snapshot management, and migration SQL generation for HowlerOps.

## Features

- Compare schemas between two live database connections
- Compare a live connection against a saved snapshot
- Save and manage schema snapshots
- Generate DDL migration scripts (PostgreSQL, MySQL, SQLite)
- Support for tables, columns, indexes, and foreign keys
- Detailed diff summaries and statistics

## Architecture

The package consists of four main components:

### 1. Types (`types.go`)

Core data structures for representing schema differences:

- `SchemaDiff` - Complete diff result with summary and detailed changes
- `TableDiff` - Diff for a single table
- `ColumnDiff`, `IndexDiff`, `FKDiff` - Diffs for specific schema objects
- `SchemaSnapshot` - Point-in-time snapshot of database schema
- `DiffStatus` - Status constants (added, removed, modified, unchanged)

### 2. Comparator (`comparator.go`)

Schema comparison logic:

```go
comparator := schemadiff.NewComparator()

// Compare two live connections
diff, err := comparator.CompareConnections(ctx, sourceDB, targetDB, "source-id", "target-id")

// Compare live connection against snapshot
diff, err := comparator.CompareWithSnapshot(ctx, liveDB, "live-id", snapshot)
```

### 3. Snapshot Manager (`snapshot.go`)

Snapshot persistence and management:

```go
manager, err := schemadiff.NewSnapshotManager("/path/to/snapshots")

// Save a snapshot
snapshot, err := manager.SaveSnapshot(ctx, db, "connection-id", "Production 2024-01-15")

// Load a snapshot
snapshot, err := manager.LoadSnapshot("snapshot-id")

// List all snapshots
snapshots, err := manager.ListSnapshots()

// Delete a snapshot
err := manager.DeleteSnapshot("snapshot-id")
```

Snapshots are stored as JSON files with SHA-256 hash verification.

### 4. Migration Generator (`migration.go`)

DDL migration SQL generation:

```go
generator := schemadiff.NewMigrationGenerator(database.PostgreSQL)

// Generate forward migration (source -> target)
sql, err := generator.GenerateSQL(diff, schemadiff.MigrationForward)

// Generate backward migration (target -> source)
sql, err := generator.GenerateSQL(diff, schemadiff.MigrationBackward)
```

Supported databases:
- PostgreSQL (full ALTER TABLE support)
- MySQL/MariaDB (full ALTER TABLE support)
- TiDB (PostgreSQL-compatible)
- SQLite (limited - requires table recreation for column modifications)

## Usage Examples

### Compare Two Database Connections

```go
import (
    "context"
    "github.com/jbeck018/howlerops/backend-go/pkg/database"
    "github.com/jbeck018/howlerops/backend-go/pkg/schemadiff"
)

// Create database connections
prodDB, err := database.NewPostgresDatabase(prodConfig, logger)
stagingDB, err := database.NewPostgresDatabase(stagingConfig, logger)

// Compare schemas
comparator := schemadiff.NewComparator()
diff, err := comparator.CompareConnections(ctx, prodDB, stagingDB, "prod", "staging")

// Check summary
fmt.Printf("Tables added: %d\n", diff.Summary.TablesAdded)
fmt.Printf("Tables removed: %d\n", diff.Summary.TablesRemoved)
fmt.Printf("Tables modified: %d\n", diff.Summary.TablesModified)
fmt.Printf("Columns changed: %d\n",
    diff.Summary.ColumnsAdded + diff.Summary.ColumnsRemoved + diff.Summary.ColumnsModified)
```

### Save and Compare Against Snapshot

```go
// Save production snapshot
manager, err := schemadiff.NewSnapshotManager("/app/data/snapshots")
snapshot, err := manager.SaveSnapshot(ctx, prodDB, "prod-conn", "Pre-deployment snapshot")

// Later... compare current state against snapshot
currentDB, err := database.NewPostgresDatabase(prodConfig, logger)
comparator := schemadiff.NewComparator()
diff, err := comparator.CompareWithSnapshot(ctx, currentDB, "current", snapshot)

// Check what changed since snapshot
for _, tableDiff := range diff.Tables {
    if tableDiff.Status == schemadiff.DiffModified {
        fmt.Printf("Table %s.%s was modified\n", tableDiff.Schema, tableDiff.Name)
        fmt.Printf("  Columns changed: %d\n", len(tableDiff.Columns))
    }
}
```

### Generate Migration SQL

```go
// Compare schemas
comparator := schemadiff.NewComparator()
diff, err := comparator.CompareConnections(ctx, devDB, prodDB, "dev", "prod")

// Generate migration to apply dev changes to prod
generator := schemadiff.NewMigrationGenerator(database.PostgreSQL)
sql, err := generator.GenerateSQL(diff, schemadiff.MigrationForward)

fmt.Println(sql)
// Output:
// -- Migration generated by HowlerOps
// -- Source: dev
// -- Target: prod
// -- Direction: forward
// ...
// ALTER TABLE public.users ADD COLUMN email_verified boolean DEFAULT false;
// CREATE INDEX idx_users_email ON public.users (email);
```

## Implementation Details

### Comparison Algorithm

1. **Schema Introspection**: Uses existing `database.Database` interface methods:
   - `GetSchemas(ctx)` - List all schemas
   - `GetTables(ctx, schema)` - List tables per schema
   - `GetTableStructure(ctx, schema, table)` - Get detailed structure

2. **Snapshot Creation**: Captures complete schema state including:
   - All schemas and tables
   - Complete table structures (columns, indexes, foreign keys)
   - Metadata and statistics

3. **Comparison Logic**:
   - Build maps of source and target schemas for efficient lookup
   - Compare each table by schema.table key
   - Detect added, removed, and modified objects
   - Generate detailed diffs for modified objects

4. **Summary Generation**: Aggregate statistics for quick overview

### Snapshot Storage

Snapshots are stored as JSON files in the configured directory:

```
/snapshots/
├── a1b2c3d4-e5f6-7890-abcd-ef1234567890.json
├── b2c3d4e5-f6a7-8901-bcde-f12345678901.json
└── ...
```

Each snapshot includes:
- Unique ID (UUID)
- Human-readable name
- Connection metadata
- Complete schema structures
- SHA-256 hash for verification
- Timestamp

### Migration SQL Generation

The migration generator produces database-specific DDL:

**PostgreSQL/TiDB:**
- Full ALTER TABLE support
- Proper identifier quoting with `"`
- Column type changes, nullability, defaults
- Index and foreign key modifications

**MySQL/MariaDB:**
- Full ALTER TABLE support
- Identifier quoting with backticks
- Similar to PostgreSQL syntax

**SQLite:**
- Limited ALTER TABLE support
- Warns about table recreation requirement
- Provides guidance on manual migration

## Error Handling

All operations return errors for:
- Database connection failures
- Schema introspection errors
- Snapshot I/O errors
- Hash verification failures
- Invalid snapshot formats

Always check errors:

```go
diff, err := comparator.CompareConnections(ctx, db1, db2, "1", "2")
if err != nil {
    return fmt.Errorf("schema comparison failed: %w", err)
}
```

## Performance Considerations

- **Large schemas**: Comparison time scales with number of tables and complexity
- **Caching**: Uses existing `database.Database` schema caching where available
- **Snapshots**: JSON files are human-readable but can be large for huge schemas
- **Memory**: Entire schemas loaded into memory for comparison

For schemas with 200+ tables, expect:
- Comparison: 3-5 seconds
- Snapshot save/load: 1-2 seconds
- Migration generation: < 1 second

## Integration with Wails

This package is designed to be called from Wails app.go bindings:

```go
// In app.go
func (a *App) CompareSchemas(sourceSessionID, targetSessionID string) (*schemadiff.SchemaDiff, error) {
    sourceDB := a.getDatabase(sourceSessionID)
    targetDB := a.getDatabase(targetSessionID)

    comparator := schemadiff.NewComparator()
    return comparator.CompareConnections(context.Background(), sourceDB, targetDB, sourceSessionID, targetSessionID)
}

func (a *App) SaveSchemaSnapshot(sessionID, name string) (*schemadiff.SchemaSnapshot, error) {
    db := a.getDatabase(sessionID)
    manager, _ := schemadiff.NewSnapshotManager(a.snapshotDir)
    return manager.SaveSnapshot(context.Background(), db, sessionID, name)
}
```
